the task

given a huge text, extract all words, creates abbreviations/acronyms for (perhaps some or majority of) the words to reduce the amount of typing a human has to do at the computer.

we also may need to generate acronyms for sentences and not just words, consider "what the fuck" -> "wtf".

current methods

preexisting compression methods like huffman coding dont serve us well, because they dont preserve the familiarity of the word, we need something that shortens words into recognizable abbreviations that are easy to remember.

ideas

  • generate a small transformer to generate abbreviations for words (perhaps we could find some dataset). but what do we do if it generates the same abbreviation for different words?
  • what does it mean for a word to retain meaningfulness? perhaps the most prominent characters should be extracted for that to happen, a prominent character is one without which the word would lose its recognizability.
  • we could adopt a measure of similarity (or a measure of meaning-preservation) between two words, then, given one word, we iterate through all permutations of length up to a specific constant and grab the top word (or top few words) we need, maybe we should adopt a specific tradeoff between length and similarity since a -length may be a better option than a -length string even if has a higher score of similarity with the target word (the goal is to shorten strings, after all).

using machine learning

we could train a transformer on a set of abbreviations, a transformer is a sequence to sequence model that can generate multiple sequences with different probabilities, we can use those probabilities and give the most frequent word in text the best acronym ("best" as in most probable or shortest, we may define different criteria).

this could work but it has its limitations.

simpler techniques

we could take the first letter of every word in the sequence (if it is a sentence), but this may not generalize well as many different sentences may correspond to the same abbreviation, and a machine learning model could learn more complicated patterns than "grabbing the first letter".

what about words? things like "background" -> "bg", how do we tell which letters are most important? we could use letter frequency distribution to extract the most important letters, which would probably correspond to the least frequent letters in the frequency table, because having an "x" in the acronym (if it exists in the word) is more important than having an "a", since many words contain "a" so the human mind may associate less words and thus less acronyms with the letter "x".

sometimes its best to shorten a word to its prefix, but this will generalize the worst, since many words may have the same prefix.

entropy

password entropy measures the unpredicatability of a word, perhaps we want the inverse of that? we want words that are most predictable, we could use entropy as a criterion here.

reduction size

a technique should be evaluated based on how much compression it succeeds to apply and how recognizable the generated abbreviations are, the former can be automated, the latter, im not yet sure.

this script lists all latex macro definitions 

#!/bin/bash

# Get the TeX Live root directory
texlive_root=$(kpsewhich article.sty | sed -r 's|/tex/latex/base/article\.sty$||')

# Get all paths from kpsewhich
paths=$(kpsewhich -show-path=tex)

# Append the automatically detected root
paths+=":$texlive_root"

# Split paths by ':' and iterate through them
IFS=':' read -ra dirs <<< "$paths"
for dir in "${dirs[@]}"; do
    # Remove leading '!!' if present
    clean_dir="${dir#!!}"

    # Skip if the directory doesn't exist
    [ -d "$clean_dir" ] || continue

    # Find .sty files in this directory
    find -L "$clean_dir" -type f -name "*.sty" -print | while read -r sty_file; do
        # Extract macros from the .sty file
        echo "$sty_file"
        grep -E '\\(new|renew|provide)command\\' "$sty_file" >> macros.txt
    done
done

from nltk.corpus import words

string = "thisisinsane"
tokens = []
lowercaseCorpus = [x.lower() for x in words.words()]
i = 0
while i < len(string):
    maxWord = ""
    for j in range(i, len(string)):
        tempWord = string[i:j+1]
        if tempWord in lowercaseCorpus and len(tempWord) > len(maxWord):
            maxWord = tempWord
    i = i+len(maxWord)
    tokens.append(maxWord)

print(tokens)
